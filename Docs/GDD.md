# 수박게임 GDD

작성일: 2026-05-12  
대상 프로젝트: `unity-ai-suika-LteFroggy`  
엔진 기준: Unity 6000.3.15f1, URP 2D, Input System

## 1. 작업 개요

개발은 아래 3단계로 진행한다.

1. 메인 메커닉
   - 보드, 과일 데이터, 조준/드롭, 머지, 점수, 게임오버, 재시작 구현.
   - 목표는 임시 그래픽만으로도 한 판이 완전히 돌아가는 상태를 만드는 것.

2. 에셋 생성 및 적용
   - 과일 11종, 보드, 배경, 위험 라인, 머지 이펙트, 기본 사운드 적용.
   - 목표는 게임 규칙이 더 잘 읽히고 머지 보상이 명확하게 느껴지는 상태를 만드는 것.

3. UI 및 마무리
   - 점수, 최고 점수, 다음 과일, 게임오버 화면, 재시작 흐름, 밸런스 튜닝, QA.
   - 목표는 새 플레이어가 바로 이해하고 반복 플레이할 수 있는 MVP 빌드로 정리하는 것.

우선순위는 반드시 `메인 메커닉 -> 에셋 -> UI/마무리` 순서로 둔다. 수박게임의 핵심 재미는 물리와 머지 체감에서 먼저 결정되므로, 첫 단계에서는 보기 좋은 결과보다 플레이 가능한 핵심 루프 완성을 우선한다.

## 2. 게임 요약

수박게임은 물리 기반 머지 퍼즐이다. 플레이어는 상자 안에 과일을 하나씩 떨어뜨리고, 같은 종류의 과일끼리 닿으면 더 큰 다음 단계 과일로 합쳐진다. 목표는 상자를 넘치지 않게 유지하면서 높은 점수를 얻고 최종 단계인 수박을 만드는 것이다.

핵심 특징:

- 시간 제한 없이 한 수씩 배치하는 낙하 퍼즐.
- 같은 과일 2개가 닿으면 다음 단계 과일 1개로 머지.
- 과일은 둥글고 물리적으로 굴러다니기 때문에 매 판 결과가 달라짐.
- 상단 위험 라인을 넘치면 게임오버.
- 다음 과일을 미리 보여줘 단기 계획을 세우게 함.

우리가 만들 MVP는 원작의 핵심 재미를 학습하고 재현하는 클론형 프로토타입이다. 단, 원작 이름, 로고, 캐릭터, 그래픽, 사운드는 그대로 사용하지 않는다.

## 3. 핵심 목표

### 3.1 한 줄 소개

귀여운 과일을 조심스럽게 떨어뜨리고 합쳐서, 상자를 넘치지 않게 유지하며 가장 큰 수박까지 성장시키는 2D 물리 머지 퍼즐.

### 3.2 핵심 재미

- 지금 나온 과일을 어디에 떨어뜨릴지 고민한다.
- 과일이 굴러가며 예상과 살짝 다르게 쌓인다.
- 같은 과일이 합쳐지며 공간이 정리되고 점수가 오른다.
- 큰 과일이 생기면서 보상과 위험이 동시에 커진다.
- 한 번 더 잘 놓으면 연쇄 머지가 될 것 같은 기대감이 생긴다.

### 3.3 디자인 원칙

- 규칙은 단순하게 유지한다.
- 물리 결과는 납득 가능해야 한다.
- 머지는 즉시 읽히고 기분 좋아야 한다.
- 게임오버는 갑작스럽기보다 이해 가능해야 한다.
- 재시작은 빠르게 가능해야 한다.
- 과일 크기, 점수, 확률은 데이터로 조정 가능해야 한다.

## 4. 게임 규칙

### 4.1 기본 루프

1. 게임 시작.
2. 현재 과일과 다음 과일이 결정된다.
3. 플레이어가 현재 과일의 x 위치를 조준한다.
4. 클릭 또는 입력으로 과일을 떨어뜨린다.
5. 같은 레벨의 과일 2개가 충돌하면 다음 레벨 과일 1개로 합쳐진다.
6. 점수가 증가한다.
7. 다음 과일이 현재 과일이 되고, 새 다음 과일이 뽑힌다.
8. 과일이 위험 라인 위에 일정 시간 머무르면 게임오버.
9. 결과 화면에서 점수와 재시작을 제공한다.

### 4.2 과일 단계

| 레벨 | 과일 | 드롭 등장 
| --- | --- | --- | 
| 1 | 체리 | 가능 | 
| 2 | 딸기 | 가능 | 
| 3 | 포도 | 가능 | 
| 4 | 귤 | 가능 | 
| 5 | 감 | 가능 | 
| 6 | 사과 | 불가 | 
| 7 | 배 | 불가 |
| 8 | 복숭아 | 불가 | 
| 9 | 파인애플 | 불가 | 
| 10 | 멜론 | 불가 | 
| 11 | 수박 | 불가 | 

MVP에서는 레벨 1~5만 랜덤 드롭 대상으로 사용한다.

### 4.3 머지 규칙

- 같은 레벨 과일 2개가 충돌하면 머지한다.
- 머지 위치는 두 과일 중심의 평균 지점으로 한다.
- 기존 과일 2개는 제거하고 다음 레벨 과일 1개를 생성한다.
- 머지 점수를 지급한다.
- 같은 과일이 한 프레임에 여러 번 머지되지 않도록 중복 방지 플래그를 둔다.
- 수박 2개가 머지되면 새 과일을 만들지 않고 둘 다 제거하며 보너스 점수를 지급한다.

### 4.4 게임오버 규칙

- 보드 상단에 위험 라인을 둔다.
- 과일이 위험 라인 위에 올라간 즉시 종료하지 않는다.
- 과일이 위험 라인 위에 `1.0초` 이상 머무르면 게임오버 처리한다.
- 머지 직후 순간적으로 튀는 경우를 줄이기 위해 짧은 유예를 둔다.
- 게임오버 후에는 추가 드롭과 머지 처리를 막는다.

## 6. 입력과 UX

### 6.1 입력

PC 기준:

- 마우스 이동: 드롭 위치 조준
- 왼쪽 클릭 또는 Space: 현재 과일 드롭
- R: 재시작
- Esc: 일시정지

드롭 위치는 보드 좌우 벽 안에서만 움직인다. 현재 과일 반지름을 고려해 벽에 파묻히지 않도록 x 위치를 clamp한다.

### 6.2 화면 요소

MVP 필수 요소:

- 중앙 보드
- 현재 조준 중인 과일
- 다음 과일 표시
- 현재 점수
- 최고 점수
- 위험 라인
- 게임오버 패널
- 재시작 버튼

### 6.3 피드백

- 드롭 시 짧은 반응음.
- 머지 시 pop 애니메이션과 점수 증가 표시.
- 고레벨 머지는 더 큰 이펙트.
- 위험 라인 초과 중에는 라인을 붉게 강조.
- 게임오버 후 결과와 재시작 버튼 표시.

## 8. 작업 순서

### 8.1 메인 메커닉

1. `[FEAT] 게임 씬/보드 기본 구조 구축`
   - `Game` 씬 구성.
   - 보드 벽, 바닥, 위험 라인 배치.
   - `BoardController` 구현.

2. `[FEAT] 과일 데이터와 프리팹 구조 구현`
   - `FruitDefinition` 작성.
   - 11단계 과일 데이터 구성.
   - 공통 과일 프리팹 구성.

3. `[FEAT] 조준 및 드롭 구현`
   - 마우스 위치 기반 조준.
   - 현재 과일 preview.
   - 클릭/Space로 드롭.

4. `[FEAT] 머지 시스템 구현`
   - 같은 레벨 충돌 감지.
   - 중복 머지 방지.
   - 다음 레벨 과일 생성.

5. `[FEAT] 점수와 다음 과일 큐 구현`
   - 현재/다음 과일 큐.
   - 드롭 확률 테이블.
   - 점수와 최고 점수.

6. `[FEAT] 게임오버 및 재시작 구현`
   - 위험 라인 체류 판정.
   - 게임오버 상태.
   - 재시작 초기화.

### 8.2 에셋 생성 및 적용

1. `[ASSET] MVP 과일 스프라이트 제작`
2. `[ASSET] 보드/배경/위험 라인 에셋 제작`
3. `[ASSET] 머지/점수/게임오버 피드백 적용`
4. `[ASSET] 기본 사운드 적용`

### 8.3 UI 및 마무리

1. `[FEAT] 플레이 HUD 구현`
2. `[FEAT] 게임오버 화면 구현`
3. `[BALANCE] MVP 밸런스 튜닝`
4. `[BUG] 플레이 테스트 및 버그 수정`

## 9. MVP 완료 기준

MVP는 다음 조건을 만족하면 완료로 본다.

- Unity 에디터에서 Play를 누르면 바로 게임이 시작된다.
- 마우스로 조준하고 클릭으로 과일을 떨어뜨릴 수 있다.
- 과일이 물리적으로 쌓인다.
- 같은 과일끼리 충돌하면 다음 과일로 합쳐진다.
- 점수가 올라간다.
- 다음 과일이 보인다.
- 과일이 상단 위험 라인을 넘고 일정 시간 지나면 게임오버가 된다.
- 게임오버 후 재시작할 수 있다.
- 임시 또는 최종 에셋으로 과일 단계가 구분된다.

## 10. 현재 결정값

- 플랫폼: PC 에디터 우선, 이후 WebGL 고려.
- 입력: 마우스 조준 + 클릭 드롭 우선.
- 드롭 가능 과일: 레벨 1~5.
- 수박 2개 머지: 제거 + 보너스 점수.
- 게임오버 유예: 1.0초.
- 과일 충돌: `CircleCollider2D` 기준.
- 저장: 최고 점수는 MVP에서 `PlayerPrefs` 사용.
